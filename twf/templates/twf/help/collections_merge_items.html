Merge Collection Items

This feature allows you to combine multiple collection items into a single item. Merging is useful when you discover that separate collection items actually represent the same logical entity or when you need to consolidate related items for more efficient processing.

When to Merge Collection Items

Consider merging collection items when:

  • Multiple items represent different parts of the same document
  • You've identified duplicate items that should be consolidated
  • Related items would benefit from being processed as a single unit
  • You need to reorganize your collection structure
  • Separate items contain complementary information about the same entity

The Merging Process

Merging collection items involves several steps:

  1. Select the items you want to merge
  2. Designate a primary item that will serve as the base for the merged result
  3. Configure how metadata from different items should be combined
  4. Preview the merge result to verify it meets your needs
  5. Confirm the merge operation
  6. Review the merged item to ensure all content is properly integrated

Merge Configuration Options

When merging items, you have several configuration options:

  • Primary Item Selection: Determine which item serves as the foundation for the merge
  • Metadata Handling: Choose how conflicting metadata fields should be resolved
  • Content Ordering: Specify the sequence for combining document content
  • Annotation Integration: Configure how annotations from different items are combined
  • Conflict Resolution: Set rules for handling conflicting information

Metadata Consolidation

When merging items with different metadata values, you can choose to:

  • Use primary: Keep only the metadata from the primary item
  • Combine all: Maintain all metadata as lists or combined values
  • Keep non-conflicting: Merge metadata that doesn't conflict
  • Manual selection: Individually select which values to keep for each field
  • Create composite: Generate new metadata that synthesizes information from all sources

Important Considerations

Before merging collection items, keep these points in mind:

  • Merging is not easily reversible - consider backing up your collection first
  • All items being merged should be logically related
  • Carefully review metadata conflicts before confirming the merge
  • Consider how the merge will affect any linked dictionaries or tags
  • Verify that any document-specific configurations will remain valid after merging

Best Practices

  • Start with a well-organized collection before attempting merges
  • Merge related items in small batches rather than all at once
  • Document your merge decisions, especially for complex cases
  • Review merged items thoroughly after the operation
  • Consider how merging affects your overall collection structure
  • Update any references to the original items in external systems
