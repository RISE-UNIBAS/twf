User Management

The User Management view allows administrators and staff users to manage user accounts, including creating new users, resetting passwords, deactivating, and deleting accounts. This view is only available to users with administrative privileges.

This is the user management for TWF. Users are created and managed here. After creating a user, you can add them to projects in the project user management view and assign them permissions.

Be careful when deleting or deactivating users, as this may affect project permissions.

Creating a New User

To create a new user, enter a username and an optional email address, then click "Create User". The user will be added to the system with default permissions.

Managing Existing Users

The list of existing users appears below the "Create New User" section. Each user has the following options:

  • Reset Password: Sends an email to the user with a new randomly generated password.
  • Activate/Deactivate: If a user is deactivated, they cannot log in and their project and their project permissions are removed. Deactivating a user does not delete the account or user activity history. Activating restores login access. Project permissions must be reset.
  • Delete: Permanently removes the user from the system. All user-related information is deleted. This action cannot be undone.

User Status and Badges

Each user may have special badges indicating their status:

  • admin - This user all has administrative privileges.
  • staff - This user has administrative privileges.
  • you - This badge appears next to your own account. You cannot delete or deactivate your own account. If you need to reset your password, use the "Forgot Password" link on the login page.
  • inactive - This user is currently deactivated and cannot log in.

Note: Deleting or deactivating a user may affect project access and permissions.
