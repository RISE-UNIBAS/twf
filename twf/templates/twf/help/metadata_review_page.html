Review Page Metadata

This page allows you to review, edit, and manage metadata specific to individual pages within a document. Page-level metadata captures information that may vary from page to page, providing granular control and description.

Page Metadata Overview

Page-level metadata can include:

  • Page numbers and sequencing information
  • Image quality and technical metadata
  • Page type (title page, index, body, appendix, etc.)
  • Content-specific information unique to the page
  • Transcription quality and status
  • Physical characteristics or condition notes
  • Layout information and structural elements

Page Metadata vs Document Metadata

While document metadata applies to the entire document, page metadata:

  • Is specific to a single page within the document
  • Can vary from page to page
  • Often includes more technical or structural details
  • May complement document-level information with page-specific context
  • Can indicate processing status or quality at the page level

Review and Editing Functions

On this page, you can:

  • View all metadata associated with a specific page
  • Edit existing metadata fields
  • Add new metadata specific to this page
  • Remove incorrect or irrelevant metadata
  • Navigate between pages while reviewing metadata
  • Compare page metadata with document metadata
  • Save changes to individual fields or all at once

Common Page Metadata Fields

Frequently used page metadata includes:

  • Page Number: Both the physical/printed number and the sequential position
  • Page Type: Classification of the page's function or content type
  • Image Parameters: Resolution, color depth, file format, etc.
  • Processing Status: Transcription status, review status, etc.
  • Special Content: Presence of illustrations, tables, marginalia, etc.
  • Technical Notes: Information about scanning, OCR quality, or issues

Best Practices

  • Be consistent in how you apply page metadata across your collection
  • Focus on metadata that will be useful for search, navigation, or analysis
  • Consider using batch operations for metadata that applies to multiple pages
  • Document any special cases or exceptions in your methodology
  • Prioritize accuracy over completeness if resources are limited
  • Review page metadata in context with the actual page content
