Export Project

The Export Project page allows you to create a comprehensive export of your entire project, including all documents, collections, dictionaries, tags, and metadata. This is useful for project backup, archiving, publication, sharing with collaborators, or transferring to another system.

Export Process

  1. Configure Project Export:
    • Choose which components to include (documents, collections, dictionaries, tags, etc.)
    • Set metadata fields to include in the export
    • Configure export format and structure
    • Set up file naming conventions
    • Configure inclusion of related data
  2. Format Options: Select from available export formats:
    • JSON (comprehensive data preservation with full structure)
    • XML (structured format with good interoperability)
    • CSV (tabular data for dictionaries and metadata)
    • PDF (human-readable documents and reports)
    • ZIP (bundled file collection with preserved directory structure)
    • TEI (Text Encoding Initiative format for scholarly texts)
    • Custom formats configured in project settings
  3. Data Selection: Refine what specific content should be included:
    • All project data or selective content
    • Include/exclude specific document types
    • Filter by dates, metadata values, or other criteria
    • Include/exclude system metadata
    • Set depth of related content inclusion
    • Configure handling of unresolved tags or incomplete entries
  4. Output Configuration:
    • Set file organization structure
    • Configure file naming patterns
    • Add supplementary files (README, data dictionary, etc.)
    • Include project configuration for migration purposes
    • Add version information and export timestamp
  5. Start Export: Begin the export process with your selected configuration.
  6. Monitor Progress: For large projects, you can track the export progress.
  7. Download or Share: Once complete, download the export or share it directly.

Project Export Content

A comprehensive project export typically includes:

  • Project Metadata: Title, description, owners, settings, version, keywords, license information
  • Documents: All document content, structure, transcriptions, and metadata
  • Pages: Page content, images, transcriptions, and page-specific metadata
  • Collections: All collections, their structure, items, and metadata
  • Dictionaries: All dictionaries, entries, normalization data, and related metadata
  • Tags: All tag data across the project, including status and relationships
  • Workflows: Information about established workflows and their configurations
  • User Data: Relevant user information (limited for privacy considerations)
  • Export Configuration: Details about how the export was generated
  • Relationships: Connections between different data objects

When to Use Project Export

  • Project Backup: Create comprehensive backups of your project at key milestones
  • Project Archiving: Prepare your project for long-term storage and preservation
  • Data Publication: Export project data for formal academic publication
  • System Migration: Move your project to a different TWF instance or system
  • Data Preservation: Ensure long-term access to your research data
  • Collaboration: Share project data with researchers using different systems
  • Analysis: Export data for processing in specialized analytical tools
  • Reporting: Generate comprehensive project documentation and reports

Export Types

The system supports several export profiles:

  • Full Project: Complete export of all project components
  • Research Data: Focused on content and metadata for research use
  • Migration: Optimized for transferring to another TWF instance
  • Publication: Structured for academic publication requirements
  • Archival: Configured for long-term preservation
  • Custom: User-defined export configurations

Important Considerations

  • Project exports may be large and take significant time to process
  • Consider privacy and data protection when exporting user-related information
  • For very large projects, consider breaking exports into smaller logical components
  • Test your export configuration with a small project before exporting large datasets
  • Document your export configuration for reproducibility
  • Consider storage requirements for the exported files
  • Verify export integrity after completion
  • For migration purposes, ensure version compatibility
