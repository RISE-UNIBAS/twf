Context Help: Export Project

The Export Project page allows you to create a comprehensive export of your entire project, including all documents, collections, dictionaries, and metadata. This is useful for project backup, archiving, publication, or transferring to another system.

Export Process

  1. Configure Project Export:
    • Choose which components to include (documents, collections, dictionaries, etc.)
    • Set metadata fields to include
    • Configure export format and structure
  2. Format Options: Select from available export formats:
    • JSON (comprehensive data preservation)
    • ZIP (bundled file collection)
    • Custom formats configured in project settings
  3. Data Selection: Refine what specific content should be included:
    • All project data or selective content
    • Include/exclude specific document types
    • Include/exclude system metadata
  4. Start Export: Begin the export process with your selected configuration.

Project Export Content

A comprehensive project export typically includes:

  • Project Metadata: Title, description, owners, settings, etc.
  • Documents: All document content, structure, and metadata.
  • Pages: Page content, images, and transcriptions.
  • Collections: All collections and their items.
  • Dictionaries: All dictionaries and their entries.
  • Tags: All tag data across the project.
  • Workflows: Information about established workflows.
  • User Data: Relevant user information (limited for privacy).

When to Use Project Export

  • Project Backup: Create comprehensive backups of your project.
  • Project Archiving: Prepare your project for long-term storage.
  • Data Publication: Export project data for formal publication.
  • System Migration: Move your project to a different TWF instance.
  • Data Preservation: Ensure long-term access to your research data.

Important Considerations

  • Project exports may be large and take time to process.
  • Consider privacy and data protection when exporting user-related information.
  • For very large projects, consider breaking exports into smaller components.
  • Test your export configuration with a small project before exporting large datasets.
