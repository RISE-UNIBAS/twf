Export Dictionaries

The Export Dictionaries page allows you to export dictionaries from your project in various formats. Dictionaries are collections of terms, definitions, normalized values, and external references that help standardize entities across your project and link them to authority records.

Export Process

  1. Select Dictionaries: Choose which dictionaries to include in the export:
    • Select all dictionaries or specific ones
    • Filter dictionaries by type (person, place, organization, etc.)
    • Filter dictionaries by usage in documents or collections
    • Filter by creation date or last modification
    • Filter by specific attributes or properties
  2. Choose Format: Select the desired export format:
    • JSON (comprehensive format preserving all relationships)
    • CSV (tabular format for spreadsheet use)
    • Excel (native spreadsheet format with multiple sheets)
    • XML (structured format with detailed schema)
    • RDF (linked data format for semantic web applications)
    • SKOS (Simple Knowledge Organization System format)
    • TEI (Text Encoding Initiative format for humanities scholarship)
    • Custom formats configured in your project settings
  3. Configure Options: Set specific options for the selected format:
    • Include/exclude dictionary metadata
    • Include/exclude variations and normalized forms
    • Configure handling of external references and identifiers
    • Set field mappings for columnar formats
    • Configure relationship representation
    • Set character encoding and special character handling
    • Configure hierarchical structure options
    • Include/exclude usage statistics and examples
  4. Preview: Generate a sample of your export to verify format and content
  5. Start Export: Click the export button to begin the export process
  6. Monitor Progress: For large dictionaries, track the progress of the export
  7. Download: Save the completed export to your local system or server

Dictionary Content Components

When exporting dictionaries, you can include various components:

  • Dictionary Metadata: Information about the dictionary itself
    • Type and classification
    • Description and purpose
    • Creation and modification dates
    • Author and contributor information
    • Version information
    • Project association
  • Dictionary Entries: The core entries in the dictionary
    • Entry labels and preferred forms
    • Entry types and classifications
    • Definitions and descriptions
    • Creation and modification information
    • Internal identifiers
  • Variations: Different forms or spellings of each entry
    • Orthographic variants
    • Historical forms
    • Abbreviations
    • Alternative spellings
    • Multilingual equivalents
  • Normalized Values: Standardized forms for each entry
    • Preferred canonical forms
    • Standardized date formats
    • Normalized geographic coordinates
    • Standardized name formats
  • External References: Links to external authority systems
    • GND (Integrated Authority File) identifiers
    • Wikidata Q-numbers
    • Geonames identifiers
    • VIAF (Virtual International Authority File) IDs
    • Library of Congress identifiers
    • Custom external references
  • Relationships: Connections between dictionary entries
    • Hierarchical relationships (broader/narrower)
    • Associative relationships (related terms)
    • Equivalence relationships (synonyms)
    • Part-whole relationships
  • Usage Information: Data about how entries are used
    • Frequency statistics
    • Document references
    • Context examples
    • Co-occurrence patterns

Format Considerations

Different export formats have different strengths:

  • JSON:
    • Preserves complex structures and relationships
    • Ideal for programmatic processing
    • Good for complete dictionary backups
    • Suitable for reuse in other TWF projects
  • CSV/Excel:
    • Easy to view and edit in spreadsheet applications
    • Good for manual review and editing
    • Works well for simpler dictionary structures
    • Limited in representing complex relationships
    • May require multiple files/sheets for complete representation
  • XML/TEI:
    • Highly structured with validation capabilities
    • Good for scholarly exchange and publication
    • Compatible with many digital humanities tools
    • Excellent for detailed documentation
  • RDF/SKOS:
    • Optimized for linked data and semantic web applications
    • Excellent for integration with other knowledge bases
    • Supports sophisticated querying capabilities
    • Ideal for vocabulary sharing across institutions

Use Cases for Dictionary Exports

  • Data Sharing: Share standardized terminology with colleagues or other projects
  • Backup and Preservation: Create backups of your dictionaries for safekeeping
  • External Enrichment: Export to enrich the data in external tools before re-importing
  • Publication: Include standardized dictionaries as part of research data publication
  • Integration: Connect your dictionaries with external systems and knowledge bases
  • Analysis: Perform statistical analysis on dictionary content and usage patterns
  • Vocabulary Building: Contribute to shared controlled vocabularies in your field
  • Migration: Move dictionary data between different systems or platforms
  • Quality Control: Review dictionary completeness and consistency
  • Knowledge Graph Creation: Use as building blocks for semantic knowledge networks

Best Practices

  • Select the appropriate export format based on your intended use
  • Include sufficient metadata to ensure your dictionaries can be properly interpreted
  • Document any specialized structures or conventions in your dictionary
  • Consider including both human-readable labels and machine-processable identifiers
  • Preserve external reference links whenever possible
  • Test your export with a small dictionary before exporting large datasets
  • Verify the integrity and completeness of your export
  • Include version information and export date
