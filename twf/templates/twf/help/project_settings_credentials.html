Credential Settings

The Credential Settings page allows you to configure access credentials for various external services used throughout the application. These settings are essential for enabling AI-powered features, data retrieval, and integration with external platforms.

Tab Navigation

The settings are organized into tabs for each service. Click on a tab to view and configure the credentials for that specific service. Your tab selection will be preserved if you navigate away and return to this page.

Transkribus

  • Username: Your Transkribus platform username.
  • Password: Your Transkribus platform password.
  • Purpose: These credentials enable the system to connect to Transkribus to import your documents and their transcriptions.

OpenAI

  • API Key: Your OpenAI API key, obtained from the OpenAI developer dashboard.
  • Default Model: The name of the OpenAI model to use (e.g., "gpt-4", "gpt-3.5-turbo").
  • Purpose: These credentials enable AI-powered text analysis, extraction, and normalization using OpenAI's language models.

Google (Gemini)

  • API Key: Your Google API key with access to Gemini models.
  • Default Model: The name of the Gemini model to use (e.g., "gemini-pro").
  • Purpose: These credentials enable AI-powered text analysis using Google's Gemini models.

Anthropic (Claude)

  • API Key: Your Anthropic API key, obtained from the Anthropic console.
  • Default Model: The name of the Anthropic model to use (e.g., "claude-3-opus-20240229").
  • Purpose: These credentials enable AI-powered text analysis using Anthropic's Claude models.

Mistral

  • API Key: Your Mistral AI API key.
  • Default Model: The name of the Mistral model to use (e.g., "mistral-large-latest").
  • Purpose: These credentials enable AI-powered text analysis using Mistral's language models.

Geonames

  • Username: Your Geonames account username.
  • Purpose: This credential enables location normalization through the Geonames service, which helps standardize geographical references in your documents.

Zenodo

  • Access Token: Your Zenodo repository access token.
  • Purpose: This credential enables the system to export and publish your project data to Zenodo, an open science repository.

Security Considerations

  • All API keys and credentials are securely stored and only used for the specified services.
  • Your credentials are never shared with other users or projects.
  • Consider using environment-specific API keys that are limited to only the necessary permissions.

Save Changes

Click the "Save Settings" button at the bottom of the form to apply your changes. If you navigate away without saving, your changes will be lost.

Note: Most features that require external services will be unavailable until you configure the respective credentials. You only need to configure the services you intend to use.
