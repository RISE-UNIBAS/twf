Context Help: Tag Grouping

The Tag Grouping page allows you to organize related tags together, standardizing variations and creating connections between similar entities. This helps clean up your data and improve analysis by reducing redundancy.

What is Tag Grouping?

Tag grouping is the process of:

  • Identifying variations of the same entity (e.g., different spellings of a name)
  • Connecting these variations to a standardized form
  • Creating relationships between tags that represent the same real-world entity
  • Reducing redundancy in your tag collection

Grouping Process

  1. Tag Selection: Choose a tag type to work with (person, place, organization, etc.)
  2. View Similar Tags: The system suggests potentially related tags based on:
    • Text similarity
    • Context patterns
    • Dictionary associations
  3. Create Groups: Select tags that should be grouped together
  4. Choose Primary Form: Designate one form as the standard representation
  5. Confirm Grouping: Apply the grouping to connect the selected tags

Grouping Methods

TWF offers several approaches to tag grouping:

  • Manual Grouping: Individually select and group related tags
  • Similarity-Based Suggestions: System suggests potential matches based on text similarity
  • Dictionary-Based Grouping: Use dictionary entries to standardize variations
  • Batch Grouping: Process multiple potential groups at once

Benefits of Tag Grouping

  • Improved Data Quality: Standardize variations and reduce duplicates
  • Better Analysis: Get more accurate counts and distributions
  • Enhanced Searchability: Find all instances of an entity regardless of variation
  • Simpler Visualization: Create clearer networks and relationships
  • Consistent Exports: Generate standardized data for publishing

Best Practices

  • Work systematically by tag type or frequency
  • Start with the most common variations
  • Be careful with ambiguous names that might refer to different entities
  • Consider context when determining if tags represent the same entity
  • Use dictionaries to validate grouping decisions when possible
  • Document your grouping decisions for consistency
