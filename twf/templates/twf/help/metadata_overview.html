Metadata Management

The Metadata section allows you to manage, edit, and organize information about your documents and pages. Effective metadata management is essential for document discovery, organization, and analysis.

What is Metadata?

Metadata is structured information that describes your documents. It includes:

  • Descriptive information (title, author, subject, etc.)
  • Technical details (file format, creation date, page count, etc.)
  • Administrative information (rights, permissions, preservation notes, etc.)
  • Structural metadata (organization, relationships between documents, etc.)
  • Content-based information (keywords, summaries, classifications, etc.)

Metadata Management Features

In the Metadata section, you can:

  • Import metadata from JSON files or Google Sheets
  • Extract metadata automatically using AI analysis
  • Edit metadata at both document and page levels
  • Review and validate metadata for accuracy and completeness
  • Normalize metadata values for consistency
  • Manage metadata fields and configure metadata schemas

Metadata Levels

The system supports metadata at different levels:

  • Project metadata: Information about the entire collection
  • Document metadata: Information about specific documents
  • Page metadata: Information about individual pages

Each level has its own set of appropriate metadata fields and management interfaces.

Why Metadata Matters

Good metadata management provides several benefits:

  • Improved discoverability and searchability of documents
  • Better organization and classification of your collection
  • Enhanced context for understanding document content
  • Support for data analysis and visualization
  • Facilitation of data exchange and interoperability
  • Preservation of essential information about your documents

Best Practices

  • Develop a consistent metadata schema for your project
  • Document your metadata practices and standards
  • Balance comprehensiveness with practicality
  • Use controlled vocabularies where appropriate
  • Implement quality control processes for metadata
  • Consider future uses when designing metadata fields
