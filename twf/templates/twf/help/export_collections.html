Export Collections

The Export Collections page allows you to export collections from your project in various formats. Collections organize related documents, pages, or annotations into meaningful groups that share common characteristics, research purposes, or project workflows.

Export Process

  1. Select Collections: Choose which collections to include in the export:
    • Select individual collections or multiple collections
    • Filter collections by creation date, author, or status
    • Filter by collection size or content type
    • Include/exclude collection hierarchies
    • Set inclusion depth for nested collections
  2. Choose Format: Select the desired export format:
    • JSON (comprehensive format with full structure preservation)
    • XML (structured format with schema options)
    • CSV (tabular format for collection metadata)
    • ZIP (bundled archive of collection contents)
    • TEI (Text Encoding Initiative format for scholarly use)
    • PDF (human-readable document format)
    • Custom formats configured in project settings
  3. Configure Options: Set specific options for the selected format:
    • Include/exclude collection items' full content
    • Include/exclude metadata at collection and item levels
    • Include/exclude annotations and tags
    • Configure document content representation
    • Set file organization and naming patterns
    • Configure character encoding and special character handling
    • Set levels of detail for included components
    • Configure export of related dictionaries and tags
  4. Preview: Generate a sample of the export format to verify settings
  5. Start Export: Click the export button to begin the export process
  6. Monitor Progress: For large collections, track the export progress
  7. Download: Save the completed export to your local system or server

Collection Content Components

When exporting collections, you can include various components:

  • Collection Metadata: Information about the collection itself
    • Title and description
    • Creation and modification dates
    • Creator and contributor information
    • Purpose and scope statements
    • Classification and categorization
    • Project association
    • Version information
    • Access rights and permissions
  • Collection Structure: Organization of the collection
    • Hierarchical structure
    • Item ordering and sequence information
    • Grouping and sub-collection information
    • Relationships between collection items
  • Collection Items: Content included in the collection
    • Document references or complete content
    • Page references or complete content
    • Item-specific metadata
    • Status and workflow information
    • Item annotations and notes
  • Document Content: Actual document data
    • Full text transcriptions
    • Page images (with configurable resolution)
    • Structural markup
    • Formatting information
  • Tags and Annotations: Markup within collection items
    • Entity tags (people, places, etc.)
    • Structural annotations
    • Commentary and notes
    • Links to dictionary entries
    • Tag status information
  • Workflow Information: Processing state and history
    • Current workflow stage
    • Processing history
    • Review comments and notes
    • Task assignments

Export Format Considerations

Different formats offer different advantages:

  • JSON:
    • Preserves collection structure, relationships, and all metadata
    • Maintains hierarchical organization
    • Suitable for programmatic processing
    • Good for complete collection backup and transfer
    • Compatible with web APIs and services
  • XML/TEI:
    • Highly structured with schema validation
    • Excellent for scholarly interchange and publication
    • Good for detailed structural representation
    • Compatible with digital humanities tools and workflows
    • Supports detailed markup and annotation representation
  • CSV:
    • Simplified tabular format for collection metadata
    • Easily imported into spreadsheet applications
    • Good for overview and statistical analysis
    • Limited in representing complex structures
  • ZIP/Archive:
    • Bundles complete collection with full content and structure
    • Preserves original document formats
    • Maintains folder structure for organization
    • Includes metadata files alongside content
  • PDF:
    • Creates human-readable presentation
    • Good for publication and sharing
    • Maintains visual representation of documents
    • Limited machine processability

Use Cases for Collection Exports

  • Publication: Prepare collections for scholarly publication
  • Exhibition: Create digital exhibitions from curated collections
  • Archiving: Preserve collections as complete units for long-term storage
  • Sharing: Distribute collections to colleagues or other researchers
  • Analysis: Export collections for specialized analytical tools
  • Teaching: Create educational resources from collection materials
  • Migration: Move collections between different systems or platforms
  • Backup: Create comprehensive backups of important collection data
  • Portfolio: Showcase representative examples of your research materials

Best Practices

  • Select the appropriate export format based on your intended use
  • Include sufficient metadata to provide context for your collection
  • Consider creating a README file explaining the collection's purpose and structure
  • Test your export with a small collection before exporting large ones
  • Verify that document relationships are properly maintained in the export
  • Document any special features or conventions in your collection structure
  • Include collection documentation and description for others to understand your organization
  • Consider file size limitations when exporting collections with many images
