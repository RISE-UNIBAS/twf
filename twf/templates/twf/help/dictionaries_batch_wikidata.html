Context Help: Wikidata Batch Enrichment

The Wikidata Batch Enrichment page allows you to automatically enrich multiple dictionary entries with data from Wikidata, a collaborative knowledge base that serves as the central storage for structured data for Wikipedia and other Wikimedia projects.

What is Wikidata?

Wikidata is a free and open knowledge base that:

  • Contains structured data about millions of entities (people, places, concepts, etc.)
  • Provides unique identifiers (Q numbers) for each entity
  • Stores multilingual labels, descriptions, and properties
  • Links to external databases and authority files
  • Is community-maintained and constantly growing

Batch Enrichment Process

  1. Dictionary Selection: Choose the dictionary to enrich (compatible with all dictionary types)
  2. Entry Selection: Select the entries to be enriched with Wikidata:
    • Filter by entry status or characteristics
    • Choose all entries or a specific subset
  3. Enrichment Configuration: Set parameters for the batch process:
    • Search precision settings
    • Language preferences
    • Properties to import from Wikidata
    • Confidence threshold for automatic matching
  4. Start Batch Process: Begin the automated enrichment

Enrichment Results

After the batch process completes:

  • Successfully Enriched: Entries that were confidently matched and updated with Wikidata
  • Uncertain Matches: Entries with potential matches that require manual review
  • No Matches: Entries for which no suitable Wikidata records were found

Wikidata Data Fields

Depending on the entity type, Wikidata can provide:

  • For Persons: Birth/death dates, occupations, nationality, works, relationships
  • For Places: Coordinates, population, country, administrative division
  • For Organizations: Founding date, headquarters, members, type
  • For All Types: Alternate names, descriptions in multiple languages, external identifiers

Best Practices

  • Ensure dictionary entries have clear, specific names to improve match quality
  • Set appropriate language preferences based on your content
  • Start with smaller batches to test match quality before processing large sets
  • Review uncertain matches manually after batch processing
  • Consider using supervised workflows for ambiguous entries
  • Wikidata has excellent coverage of notable entities across many cultures and domains
