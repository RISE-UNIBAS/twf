Project Repository Settings

The Repository Settings page allows you to configure how your project interacts with external data repositories like Zenodo, GitHub, or institutional archives. These settings enable proper citation, sharing, and long-term preservation of your project data.

Repository Configuration

Repository settings typically include:

  • Data repository selection and connection details
  • Digital Object Identifiers (DOI) and persistent identifiers
  • Licensing and rights information
  • Version control and publication workflow settings
  • Embargo periods and access restrictions
  • Citation preferences and metadata formats

Digital Object Identifiers (DOIs)

DOIs provide permanent identification for your project and its data:

  • Register a new DOI for your project through supported repositories
  • Link to an existing DOI if your project is already registered
  • Configure DOI metadata to ensure proper attribution
  • Set up versioning to maintain DOI history

Licensing Options

Set appropriate licenses for your project data:

  • Choose from standard licenses (Creative Commons, MIT, etc.)
  • Configure custom license terms if needed
  • Set different licenses for different components if applicable
  • Specify attribution requirements

Publication Settings

Configure how your project is published to repositories:

  • Set publication workflows (manual or automated)
  • Configure versioning and release strategies
  • Define which project components are included in publications
  • Set up pre-publication validation checks
  • Configure embargo periods if applicable

Repository Integration

The system supports integration with various repositories:

  • Zenodo: For long-term academic data preservation
  • GitHub/GitLab: For code and version control
  • Institutional repositories: For university or organization archives
  • Domain-specific repositories: For field-specific data sharing

Best Practices

  • Choose repositories that align with your field's standards
  • Select licenses that match your sharing and reuse intentions
  • Configure comprehensive metadata for better discovery
  • Use persistent identifiers for all significant components
  • Test repository connections before major publishing events
  • Document repository choices and settings for project participants
  • Consider long-term preservation needs when selecting repositories
