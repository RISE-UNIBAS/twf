Project Repository Settings

The Repository Settings page allows you to configure how your project data will be published to external repositories like Zenodo. These settings define the metadata, licensing, and documentation that will accompany your published datasets.

Version

The version number for your dataset. Use semantic versioning (e.g., 1.0.0) to indicate major, minor, and patch changes to your data.

  • Best Practice: Increment the version number each time you publish an updated version of your dataset
  • Example: "1.2.3" (major version 1, minor version 2, patch 3)

License

The license that defines how others can use, share, and modify your published data. Choose the license that best aligns with your data sharing goals:

  • Creative Commons Attribution 4.0 (CC BY 4.0): Users can share and adapt the data with attribution
  • Creative Commons Attribution-ShareAlike 4.0 (CC BY-SA 4.0): Users can share and adapt the data with attribution, and must share under the same license
  • Creative Commons Zero v1.0 Universal (CC0 1.0): Waives all rights to the data (public domain)
  • MIT License: Permissive license focusing on software
  • GNU General Public License v3.0 (GPL-3.0): Copyleft license for software
  • Apache License 2.0: Permissive license with patent grant

Keywords

A list of relevant terms that describe your dataset. These keywords help others discover your data when searching repositories.

  • Format: Enter each keyword separately
  • Best Practice: Include terms related to your research domain, methodology, and content types
  • Example Keywords: "historical documents", "transcription", "16th century", "manuscripts"

Workflow Description

A detailed markdown document that describes your project methodology, data structure, and any processing steps. This documentation is crucial for others to understand how your data was created and how to use it.

Markdown Editor Features

  • Generate Default: Click the "Generate Default" button to automatically create a basic workflow description based on your project data
  • Live Preview: As you edit the markdown content, a preview will be shown to the right
  • Markdown Formatting: Use markdown syntax for headings (#, ##), lists (-, *), links ([text](URL)), and other formatting

What to Include in Your Workflow Description

  • Project Background: The context and purpose of your project
  • Data Collection Methods: How the original data was gathered
  • Processing Steps: The transformations and enrichments applied to the data
  • Data Structure: How the dataset is organized
  • Usage Guidelines: Instructions on how to work with the data
  • Known Limitations: Any caveats or issues users should be aware of

Digital Object Identifiers (DOIs)

When publishing to Zenodo, a DOI will be generated that provides permanent identification for your project:

  • DOIs ensure your dataset remains citable even if URLs change
  • The system will use the project metadata to prepare DOI registration
  • After publication, your project's DOI can be recorded in the project settings

Repository Integration

The system currently supports integration with:

  • Zenodo: An open data repository operated by CERN that enables long-term storage and sharing of research outputs

Best Practices

  • Choose a license that matches your sharing and reuse intentions
  • Use specific, relevant keywords to enhance discoverability
  • Create a comprehensive workflow description to improve reproducibility
  • Test repository connections before publishing (using the credentials in Project Credential Settings)
  • Ensure all team members are properly credited in the project metadata

Note: Repository settings are applied when publishing your data to external repositories like Zenodo. These settings don't affect how data is stored or processed within the application.
