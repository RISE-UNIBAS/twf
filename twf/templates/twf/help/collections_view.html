Collections View

The Collections View shows all collections available in your project. A collection is a curated set of document items grouped together for a specific purpose, such as thematic organization, batch processing, or export preparation.

Collections List

The main table displays all collections in your project with key information:

  • Title: The name of each collection
  • Description: A brief explanation of the collection's purpose
  • Type: The category or classification of the collection
  • Items: The number of document items contained in the collection
  • Status Distribution: Visual representation of item statuses within the collection
  • Created By: The user who created the collection
  • Created/Modified: Timestamps for collection creation and updates
  • Actions: Available operations for each collection

Navigation Controls

Above the collections table, you'll find various controls:

  • Create Collection: Start a new collection in your project
  • Search: Filter collections by name, description, or other properties
  • Filter: Narrow displayed collections by type, creator, or date range
  • Sort: Change how collections are ordered in the list
  • Pagination: Navigate through multiple pages of collections
  • Display Options: Adjust how many collections are shown per page

Collection Actions

For each collection, several actions are available through the Actions column:

  • View: Open the detailed view of the collection and its items
  • Edit: Modify the collection's properties and settings
  • Process: Apply batch operations or AI processing to the collection
  • Export: Generate an export of the collection
  • Delete: Remove the collection from your project
  • Copy: Create a duplicate of the collection

Status Indicators

Collections show aggregated status information for their items:

  • Open: Items that have not yet been fully processed
  • Reviewed: Items that have been verified and approved
  • Faulty: Items that have issues requiring attention
  • Status Progress: Visual bar showing the proportion of items in each status

Collection Types

Collections can be organized by various types according to your project's needs:

  • Thematic: Collections based on subject matter or topics
  • Chronological: Collections organized by time period
  • Workflow: Collections representing stages in a processing workflow
  • Export: Collections prepared specifically for export
  • Analysis: Collections grouped for specific analytical purposes
  • Custom: User-defined collection types specific to your project

Collection Management Tips

  • Use descriptive titles and detailed descriptions for better organization
  • Create separate collections for different research questions or themes
  • Review collection status indicators regularly to track progress
  • Use collection types consistently to maintain organizational structure
  • Export important collections for sharing or archiving
  • Delete obsolete collections to keep your project organized
