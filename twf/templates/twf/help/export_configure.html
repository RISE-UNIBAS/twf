Configure Export Formats

The Export Configuration page allows you to set up and manage global export settings for your project. These configurations define how content is exported, formatted, and structured across all export types, ensuring consistency and efficiency in your export workflows.

Configuration Categories

Export configuration covers several key areas:

  • Format Definitions: Configuring the structure and content of different export formats
  • Default Settings: Establishing project-wide defaults for export operations
  • Metadata Mapping: Defining how metadata fields are represented in exports
  • File Structure: Setting up organization and naming conventions for export files
  • Content Transformations: Configuring how content is processed during export
  • Templates: Creating reusable export templates for different purposes

Format Configuration Options

For each export format, you can configure:

  • Default Export Formats:
    • Set preferred formats for different content types (documents, collections, dictionaries, tags)
    • Configure format-specific options and parameters
    • Define format variations for different export scenarios
    • Set format priorities and fallbacks
  • Metadata Handling:
    • Specify which metadata fields to include in exports
    • Map internal field names to export field names
    • Configure metadata transformations and normalization
    • Set up metadata validation rules
    • Define required and optional metadata fields
  • Content Formatting:
    • Define how text content is formatted
    • Configure handling of special characters
    • Set up structural markup options
    • Configure image handling and inclusion
    • Set pagination and section break handling
  • File Structure:
    • Define file organization hierarchies
    • Set up file naming patterns and conventions
    • Configure bundling and archive options
    • Set up directory structures for multi-file exports
    • Define manifest and index file generation
  • Output Templates:
    • Create reusable templates for different export types
    • Define layout and styling for formatted outputs
    • Configure header, footer, and metadata placement
    • Set up document structure templates
    • Define custom transformation templates

Advanced Configuration Features

  • Schema Definitions: Configure validation schemas for structured formats like XML/JSON
  • XSLT Transformations: Set up transformation stylesheets for XML-based exports
  • Custom Scripts: Define custom processing scripts for specialized export needs
  • Conditional Logic: Configure conditional export rules based on content properties
  • Format Converters: Set up conversion paths between different formats
  • Post-Processing: Configure automatic post-processing operations
  • Validation Rules: Define validation checks for export content

Global vs. Individual Export Settings

Configurations exist at different levels:

  • Project-Level Defaults: Settings that apply to all exports in the project
  • Format-Specific Settings: Configurations specific to particular export formats
  • Content-Type Settings: Settings that apply to specific types of content
  • Template-Based Settings: Configurations defined in reusable templates
  • Individual Export Overrides: Settings specified during a particular export operation

Settings configured at the project level apply globally but can be overridden for individual exports when needed. This allows for consistent exports across the project while maintaining flexibility for special cases.

Reusable Export Templates

Creating export templates allows you to:

  • Save commonly used export configurations for reuse
  • Share standardized export settings across your team
  • Create purpose-specific export configurations (archival, publication, analysis, etc.)
  • Ensure consistency across multiple export operations
  • Quickly switch between different export strategies

Schema and Standard Compliance

For structured export formats, you can configure:

  • Compliance with standard schemas (TEI, METS, MODS, Dublin Core, etc.)
  • Custom schema extensions for project-specific needs
  • Validation rules to ensure standard compliance
  • Namespace configurations for XML-based formats
  • Standard identifiers and references

Best Practices for Export Configuration

  • Consider the needs of your target audience when selecting and configuring export formats
  • Document your export configurations for team reference and future use
  • Create purpose-specific templates for different export scenarios
  • Balance completeness with usability when configuring metadata inclusion
  • Test configurations with sample content before applying them to large exports
  • Consider long-term preservation needs when configuring archival formats
  • Align your export configurations with relevant standards in your field
  • Review and update export configurations periodically as project needs evolve
