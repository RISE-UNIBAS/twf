Context Help: Open Tags

The Open Tags page displays all tags in your project that have been extracted but not yet resolved or normalized. These tags require review and processing to connect them to dictionary entries or otherwise standardize them.

Understanding Open Tags

Open tags are:

  • Tags that have been extracted from your documents
  • Not yet connected to dictionary entries
  • Awaiting normalization or standardization
  • Requiring review and decision-making
  • Part of the active workflow for tag management

Working with Open Tags

From this page, you can:

  • View Tag Details: See the tag text and its context
  • Filter Tags: Focus on specific tag types or patterns
  • Assign to Dictionary: Connect tags to existing dictionary entries
  • Create Dictionary Entries: Add new dictionary entries for unmatched tags
  • Park Tags: Set aside tags that require further consideration
  • Ignore Tags: Mark tags as not relevant for your analysis

Tag Information

For each open tag, you can see:

  • Tag Text: The actual text of the tag
  • Tag Type: The category (person, place, organization, etc.)
  • Context: Surrounding text to help understand usage
  • Document/Page: Where the tag appears
  • Potential Matches: Suggestions for dictionary entries
  • Frequency: How often this tag appears in your documents

Processing Strategies

Consider these approaches for handling open tags:

  • Batch Processing: Handle similar tags together
  • Prioritization: Start with high-frequency tags
  • Type Focus: Process one tag type at a time
  • Dictionary Integration: Use existing dictionaries for normalization
  • AI Assistance: Leverage AI tools for suggestions

Best Practices

  • Regularly review and process open tags to prevent backlog
  • Create appropriate dictionaries before extensive tag processing
  • Use consistent criteria for resolving similar tags
  • Document decisions for special cases
  • Consider batch operations for efficiency with large tag sets
