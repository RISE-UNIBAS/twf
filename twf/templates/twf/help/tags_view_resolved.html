Resolved Tags

This page displays all tags that have been successfully resolved - meaning they have been assigned to a dictionary entry or otherwise marked as completed. Resolved tags represent the successful mapping between document tags and your project's dictionaries.

What are Resolved Tags?

Resolved tags have been:

  • Assigned to a specific dictionary entry
  • Properly categorized according to your project's taxonomy
  • Verified and confirmed as correctly identified
  • Linked to standardized or normalized entities

Working with Resolved Tags

In this view, you can:

  • View details about resolved tags
  • Filter tags by various criteria including dictionary, document, or tag type
  • Review connections between tags and dictionary entries
  • Reopen tags if you need to reassign them
  • View tag context to see where the tag appears in the document

Tag Information

For each resolved tag, you can see:

  • Tag Text: The actual text content of the tag
  • Type: The category of the tag (person, place, etc.)
  • Document: The document containing this tag
  • Dictionary: The dictionary the tag is assigned to
  • Dictionary Entry: The specific entry in the dictionary
  • Actions: Options to manage this tag

Benefits of Resolved Tags

Properly resolved tags provide several advantages:

  • Enable accurate data analysis and visualization
  • Support standardized exports and data interchange
  • Allow for consistent entity references across documents
  • Create connections between related entities in your project
  • Serve as building blocks for knowledge graphs and semantic networks
