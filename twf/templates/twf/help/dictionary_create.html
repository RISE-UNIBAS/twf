Context Help: Create New Dictionary

The Create New Dictionary page allows you to establish a new dictionary from scratch. Dictionaries in TWF are collections of terms and their standardized forms, which help normalize and enrich tags in your documents.

Dictionary Creation Form

When creating a new dictionary, you'll need to provide:

  • Dictionary Title: A clear, descriptive name for your dictionary
  • Dictionary Type: The category of terms this dictionary will contain:
    • Person: For names of individuals
    • Place: For geographic locations
    • Organization: For institutions and groups
    • Date: For date formats and references
    • General: For other types of terms
  • Description: A brief explanation of the dictionary's purpose and contents
  • Access Settings: Who can view and edit this dictionary

Dictionary Types Explained

Choosing the correct dictionary type is important:

  • Person Dictionary: Used for standardizing variations of people's names. Supports links to biographical databases like GND.
  • Place Dictionary: For geographic locations. Can be enriched with coordinates and hierarchical information from sources like GeoNames.
  • Organization Dictionary: For institutions, companies, and other formal groups. Can be linked to authority records.
  • Date Dictionary: Helps standardize different date formats and historical date references.
  • General Dictionary: For any other terms that don't fit the above categories.

After Creation

Once you create a dictionary:

  1. The empty dictionary will appear in your project's dictionary list
  2. You can begin adding entries manually or through batch processes
  3. You can enrich entries using external sources like GND, Wikidata, or GeoNames
  4. The dictionary will become available for tag normalization

Best Practices

  • Use clear, consistent naming conventions for dictionaries
  • Choose the correct type to enable appropriate enrichment options
  • Provide detailed descriptions to help other users understand the dictionary's purpose
  • Plan your dictionary structure before adding many entries
  • Consider how the dictionary will be used for tag normalization
