Editing Dictionary Entries

This page allows you to edit existing dictionary entries. Dictionary entries are the individual records within your dictionaries that represent normalized entities such as people, places, organizations, or other terms of interest.

Dictionary Entry Fields

When editing a dictionary entry, you'll work with several key fields:

  • Label: The primary identifier for this entry, typically the canonical form of a name or term
  • Type: The category this entry belongs to (e.g., person, place, organization)
  • Normalized Form: The standardized version of this entity, often linked to an external authority
  • Definition: An optional description or clarification of the entry
  • Metadata: Additional structured information about the entry
  • External Identifiers: Links to authoritative records in external systems

Editing Process

The dictionary entry editing process typically involves:

  1. Reviewing the current entry information
  2. Updating fields as needed
  3. Adding or modifying metadata
  4. Linking to external authority records
  5. Saving your changes

Working with Normalized Forms

Normalized forms are critical for data consistency. When editing an entry, you can:

  • Search external authorities like GND, Wikidata, or GeoNames
  • Select an existing normalized form from search results
  • Create a manual normalized form if no suitable match exists
  • View details about the selected normalized form
  • Change the normalized form if needed

Adding and Editing Metadata

Dictionary entries can contain rich metadata beyond the basic fields:

  • Dates of birth and death for persons
  • Geographic coordinates for places
  • Alternative names or spellings
  • Historical context information
  • Relationships to other entries
  • Custom fields specific to your project

Best Practices

  • Be consistent in how you format labels and names
  • Always link to external authorities when possible
  • Add sufficient contextual information to distinguish similar entries
  • Document your decisions about normalization
  • Consider how this entry relates to others in your project
  • Review associated document tags after making significant changes
