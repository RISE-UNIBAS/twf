Export to Zenodo

The Export to Zenodo page allows you to publish your project data to Zenodo, an open-access repository operated by CERN. This feature helps make your research data openly available, citable, and compliant with FAIR principles (Findable, Accessible, Interoperable, and Reusable).

What is Zenodo?

Zenodo is a general-purpose open repository that allows researchers to deposit research papers, data sets, research software, reports, and any other research-related digital artifacts. It offers several key benefits:

  • Assigns a DOI (Digital Object Identifier) to all publicly available uploads, making your data citable
  • Provides long-term preservation of your research data
  • Enables version control of your datasets
  • Supports various access levels from open access to restricted
  • Integrates with research workflows and other scholarly platforms
  • Complies with funding agency requirements for open data

Export Process

  1. Configure Metadata:
    • Title and description of your dataset
    • Authors and contributors with affiliations and ORCID IDs
    • Keywords, subjects, and related information
    • Choose an appropriate license
    • Set publication date and version information
    • Add funding information if applicable
  2. Select Content: Choose what to include in your Zenodo publication:
    • Documents (full texts or selected documents)
    • Collections (entire collections or specific items)
    • Dictionaries (all or selected dictionaries)
    • Project metadata and configuration
    • Tags and annotations
    • Export format configurations
  3. Export Format: Select how your data should be formatted for Zenodo:
    • JSON for machine-readable data
    • XML for structured data exchange
    • CSV for tabular data
    • PDF for human-readable documents
    • Combined packages with multiple formats
  4. Publication Type:
    • Create a new publication (new DOI)
    • Update an existing publication (same DOI, new version)
    • Create a draft for review before publishing
  5. Access Rights:
    • Open Access - freely available to all
    • Embargoed - released after a specified date
    • Restricted - limited to specified users
    • Closed - metadata visible but files not accessible
  6. Review and Submit: Verify your export settings before publishing to Zenodo.

Zenodo API Credentials

To use this feature, you need to have Zenodo API credentials configured in your project settings:

  1. Create a Zenodo account at zenodo.org (or use the Sandbox environment at sandbox.zenodo.org for testing)
  2. Generate an API token in your Zenodo account settings
  3. Add this token to your project's credential settings
  4. Test the connection before attempting a full publication

If you don't see these options, please check your project's credential settings or contact your project administrator.

Publication Tips

  • Provide thorough metadata to make your dataset more discoverable
  • Use standardized keywords and subject classifications when possible
  • Choose an appropriate license that reflects how you want others to use your data
  • Include a README file with your export that explains the dataset structure and contents
  • Consider adding a data dictionary that explains variables and their values
  • Use the Zenodo Sandbox environment for testing before doing a final publication
  • Ensure all authors and contributors are properly credited with accurate information
  • Link your dataset to related publications, if applicable
  • Add your ORCID ID to ensure proper attribution

Version Control

Zenodo supports versioning of datasets:

  • Each version receives its own DOI, but Zenodo maintains a concept DOI that always points to the latest version
  • Previous versions remain accessible and citable
  • Versioning helps track the evolution of your dataset over time
  • Consider adding version notes explaining what changed between versions
