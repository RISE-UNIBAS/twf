Context Help: Tags Overview

The Tags Overview page provides a central access point for all tag-related functions in your project. Tags are annotations within your documents that identify entities, concepts, or data points of interest.

What Are Tags?

In TWF, tags represent:

  • Named entities (people, places, organizations) in your documents
  • Dates and temporal references
  • Key terms and concepts
  • Any text that has been marked for identification and standardization

Tag Management Features

From this page, you can access different aspects of tag management:

  • Extract Tags: Identify and extract tags from document content
  • View Tags: Browse all tags across your project
  • Tag Views: Filter tags by their status (open, resolved, parked, ignored)
  • Tag Workflows: Access specialized processes for tags
  • Tag Settings: Configure tag extraction and management preferences

Tag Statistics

The overview page displays key statistics about tags in your project:

  • Total number of tags extracted
  • Distribution by tag type (person, place, date, etc.)
  • Tags by status (resolved, open, parked, ignored)
  • Recent tag activity

Tag Workflow Processes

TWF provides several specialized workflows for managing tags:

  • Tag Grouping: Organize related tags together
  • Date Normalization: Standardize date formats across your documents
  • Dictionary Linking: Connect tags to dictionary entries for standardization
  • Batch Processing: Apply operations to multiple tags at once

Tag Management Best Practices

  • Extract tags early in your project workflow
  • Create appropriate dictionaries before extensive tag normalization
  • Use tag statuses to track progress (parked for uncertain cases, resolved for completed ones)
  • Handle special tag types (like dates) with dedicated workflows
  • Review ignored tags periodically to ensure nothing important was missed
