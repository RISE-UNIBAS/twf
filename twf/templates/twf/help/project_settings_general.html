General Project Settings

The General Settings page allows you to configure the basic information and properties of your project. These settings determine how your project appears, who can access it, and what dictionaries are associated with it.

Project Information

  • Title: The name of your project. This title will be displayed throughout the application and should be descriptive and recognizable.
  • Description: A detailed explanation of your project's purpose, contents, and goals. A good description helps team members understand the project's context and objectives.

Access Control

  • Owner: The primary administrator of the project. The owner has full rights to modify all aspects of the project, including settings, data, and user access.
  • Members: Users who have access to work with the project. You can add multiple members by selecting them from the dropdown list. Members will have access based on their assigned permissions.

Resource Settings

  • Selected Dictionaries: The dictionaries associated with this project for normalization and entity recognition. You can select multiple dictionaries to use for various types of entities (persons, places, organizations, etc.).

Save Changes

Click the "Save Settings" button at the bottom of the form to apply your changes. If you navigate away without saving, your changes will be lost.

Tips for Effective Settings

  • Choose a clear, descriptive project title that reflects the content and purpose.
  • Write a comprehensive description that helps new members understand the project.
  • Carefully select the owner and members to ensure proper access control.
  • Select only the dictionaries that are relevant to your project to avoid clutter.

Note: Changes to project settings may affect how data is displayed and processed throughout the application. Review your settings carefully before saving.
