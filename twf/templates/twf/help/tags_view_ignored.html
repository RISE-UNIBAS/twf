Ignored Tags

This page shows all tags that have been marked as "ignored" - meaning they have been deliberately excluded from further processing. Ignored tags are those that you've determined are not relevant to your project's needs.

What are Ignored Tags?

Ignored tags typically include:

  • False positives from automatic tag extraction
  • Tags that fall outside your project's scope
  • Duplicates or variations that are better handled elsewhere
  • Elements that were incorrectly identified as tags
  • Tags that don't require normalization or dictionary assignment

Working with Ignored Tags

In this view, you can:

  • View details of ignored tags
  • Filter tags by various criteria including type, document, and more
  • Unignore tags if you later decide they should be processed
  • Review why certain tags were ignored
  • View tag context to see where the tag appears in the document

Tag Information

For each ignored tag, you can see:

  • Tag Text: The actual text content of the tag
  • Type: The category of the tag (person, place, etc.)
  • Document: The document containing this tag
  • Page: The page number where the tag appears
  • Actions: Options to manage this tag

When to Ignore Tags

Consider ignoring tags when:

  • The tag was incorrectly identified by automated processes
  • The tag is irrelevant to your research questions
  • The tag represents common words that don't need special handling
  • The tag would create unnecessary noise in your data analysis
  • The tag refers to entities outside your project's temporal or geographic scope
