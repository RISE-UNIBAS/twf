Creating a New Project

As an administrator, you can create a new project by clicking on the "Create Project" button in the navigation bar.

Project Details

Each project requires the following details:

  • Title: The project's title must be unique in the database.
  • Description: A brief description of the project that can be included in exports.
  • Keywords: Optional tags or keywords that help categorize your project.
  • License: The licensing terms applicable to your project data.
  • Transkribus Collection ID:
    • The ID of the collection in Transkribus, used to link the project to the collection.
    • If no collection exists in Transkribus, enter 0.
    • TWF can handle non-Transkribus data, but it is strongly recommended to start your project in Transkribus.

User Roles & Permissions

Each project has assigned roles that define user permissions:

  • Project Owner: The user assigned as the project owner. Can edit the project and manage users. Does not have to be an admin account. Permissions are assigned per project.
  • Project Members: Users who are granted access to the project. They can view the project and work within workflows. Users can be added or removed later.

Project Configuration

After creating a project, you'll need to configure several aspects:

  • API Credentials: For integration with external services like Transkribus, OpenAI, etc.
  • Export Settings: Define how project data will be exported and structured
  • Dictionaries: Create or import dictionaries for entity normalization
  • Task Configuration: Define automated task settings

Best Practices

  • Choose a descriptive project title that clearly identifies your project's purpose
  • Write a comprehensive description that explains the project goals and scope
  • Add appropriate team members early to facilitate collaboration
  • Configure API credentials right after project creation if you plan to use AI services
  • Consider creating project documentation using the notes feature
