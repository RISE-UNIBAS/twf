Review Document Metadata

This page allows you to review, edit, and manage metadata for a specific document. Document metadata provides essential contextual information and improves searchability, organization, and analysis capabilities.

Document Metadata Overview

Document-level metadata provides information about the entire document, such as:

  • Title and document identifiers
  • Author, editor, and other contributor information
  • Creation, publication, and modification dates
  • Document type and classification
  • Source information and provenance
  • Rights and usage permissions
  • Content description and keywords

Reviewing Metadata

When reviewing document metadata, you should:

  • Verify the accuracy of automatically extracted or imported metadata
  • Check for completeness and fill in any missing fields
  • Ensure consistency with your project's metadata standards
  • Look for errors or inconsistencies that need correction
  • Consider adding additional context-specific metadata

Editing Capabilities

On this page, you can:

  • Edit individual metadata fields
  • Add new metadata fields when needed
  • Remove incorrect or irrelevant metadata
  • Normalize dates, names, and other structured data
  • Link to authority records or controlled vocabularies
  • Save changes individually or as a batch

Metadata Field Types

Different types of metadata fields may be available:

  • Text fields: For titles, descriptions, and other textual information
  • Date fields: For temporal information with standardized formatting
  • Person fields: For author and contributor information, potentially linked to dictionaries
  • Location fields: For geographic information, potentially with coordinates
  • Classification fields: For document type, subject, and other categorical information
  • Identifier fields: For catalog numbers, DOIs, URIs, and other unique identifiers

Best Practices

  • Use consistent formatting across similar documents
  • Follow metadata standards relevant to your field or institution
  • Be thorough but concise in your metadata descriptions
  • Use controlled vocabularies when available
  • Document any special considerations or exceptions
  • Consider how metadata will be used in search, filtering, and exports
