Task Settings

The Task Settings page allows you to configure various task-related settings for your project. These settings control how data is processed, normalized, and validated throughout the application.

Date Normalization Settings

  • Date Input Format: Specify the format of dates in your documents. Options include:
    • Auto Detect: The system will attempt to determine the date format automatically
    • DMY: Day-Month-Year format (e.g., 25/12/2023)
    • YMD: Year-Month-Day format (e.g., 2023/12/25)
  • Resolve to Precision: Specify the level of precision to which dates should be normalized (e.g., day, month, year). This affects how partial dates are processed.

Google Sheets Connection

These settings allow you to connect your project to a Google Sheets document for importing metadata and other information.

  • Google Sheet ID: The unique identifier for your Google Sheet, found in the URL: https://docs.google.com/spreadsheets/d/GOOGLE_SHEET_ID/edit
  • Google Sheet Range: The range of cells to read from the sheet (e.g., "Sheet1!A1:Z100")
  • Google Sheet Valid Columns: A list of column names that should be processed from the sheet
  • Google Sheet Document ID Column: The column containing document identifiers to match with your project documents
  • Google Sheet Document Title Column: The column containing document titles

Metadata Review Settings

Configure how metadata is reviewed at both the page and document levels.

  • Page Metadata Review: JSON configuration specifying fields and validation rules for page-level metadata review
  • Document Metadata Review: JSON configuration specifying fields and validation rules for document-level metadata review

Format: Both fields should contain valid JSON objects that define metadata fields to review, their types, and any validation rules.

Tag Type Settings

  • Tag Type Translator: A JSON mapping that translates between different tag naming conventions. This allows standardization of tag types from different sources.
  • Ignored Tag Types: A JSON array of tag types that should be ignored during processing. Tags with these types will not be displayed or included in exports.

Working with JSON Fields

Several settings on this page require JSON data. When editing these fields:

  • Ensure your JSON is valid - use a JSON validator if needed
  • For the tag type translator, use a format like: {"original_tag_type": "standardized_tag_type"}
  • For ignored tag types, use an array format: ["tag_type_1", "tag_type_2"]

Note: Changes to task settings may affect how data is processed and displayed throughout the application. If you're uncertain about a setting, leave it at its default value.

Tip: After updating settings, perform a test operation to ensure the new configuration works as expected.
