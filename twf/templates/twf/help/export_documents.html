Export Documents

The Export Documents page allows you to export documents from your project in various formats. You can select specific documents to export, apply filters to find relevant documents, and configure detailed export settings to meet your specific needs.

Export Process

  1. Select Documents: Choose which documents to include in the export:
    • Use the table to select individual documents
    • Filter documents by metadata (title, date, author, etc.)
    • Filter by document status or workflow stage
    • Filter by tag content or dictionary associations
    • Filter by collection membership
    • Use the "Select All" option for bulk selection
    • Sort by any column to organize your selection
  2. Choose Format: Select the desired export format:
    • JSON (structured format with complete data preservation)
    • XML (structured format with schema validation)
    • TEI (Text Encoding Initiative format for humanities scholarship)
    • TXT (plain text export of document content)
    • CSV (tabular format for document metadata)
    • PDF (formatted human-readable documents)
    • HTML (web-ready document format)
    • DOCX/ODT (word processor formats)
    • Custom formats defined in your project settings
  3. Configure Options: Set specific options for the selected format:
    • Content inclusion options (full text, summaries, excerpts)
    • Page content handling (all pages, selected pages, page ranges)
    • Metadata inclusion and formatting
    • Tag and annotation handling
    • Image inclusion and resolution settings
    • File organization and naming patterns
    • Character encoding and special character handling
    • Pagination and document structure representation
    • Dictionary reference inclusion and formatting
  4. Advanced Settings: Configure additional export parameters:
    • Document bundling options (separate files vs. single archive)
    • Include supplementary files (README, metadata glossary, etc.)
    • Transformation rules for content normalization
    • Styling and formatting options for rich text formats
    • Set post-processing operations
    • Configure error handling and validation rules
  5. Preview: Generate a sample of your export for verification
  6. Start Export: Click the export button to begin the export process
  7. Monitor Progress: For large document sets, track the export progress
  8. Download: Save the completed export to your local system or server

Document Content Components

When exporting documents, you can include various components:

  • Document Metadata: Information about the document
    • Title, author, date, and other core metadata
    • Source information and provenance
    • Document classification and categorization
    • Processing status and workflow information
    • Creation and modification history
    • Project-specific metadata fields
  • Document Content: The actual content of the document
    • Full text transcriptions
    • Page images and facsimiles
    • Structural markup (headings, paragraphs, etc.)
    • Formatting information
    • Tables, lists, and other structured content
  • Pages: Page-level information
    • Page numbers and sequence information
    • Page-specific metadata
    • Image data and quality information
    • Page-specific annotations
  • Tags and Annotations: Markup within documents
    • Named entity tags (people, places, dates, etc.)
    • Dictionary references and normalizations
    • Editorial annotations and comments
    • Structural annotations
    • Tag status information
  • Relationships: Connections to other project elements
    • Collection memberships
    • Document relationships (part of, references, etc.)
    • External resource links
    • Cross-references within the project

Format Considerations

Different export formats offer various advantages:

  • JSON/XML:
    • Preserves complete document structure and metadata
    • Supports complex relationships and hierarchies
    • Machine-readable for programmatic processing
    • Good for data interchange and system integration
    • Supports complete data representation
  • TEI:
    • Specialized format for text encoding in humanities
    • Rich structural and semantic markup
    • Strong support for scholarly annotations
    • Compatible with digital humanities tools and workflows
    • Supports detailed representation of textual features
  • TXT/CSV:
    • Simple formats for basic content and metadata
    • Universal compatibility
    • Easy to process with standard tools
    • Good for simple content extraction
    • Limited in representing complex structure or formatting
  • PDF/DOCX/HTML:
    • Focused on human-readable presentation
    • Preserves visual formatting and layout
    • Good for publication and sharing
    • Widely supported viewing formats
    • May lose some structured data aspects

Use Cases for Document Exports

  • Publication: Prepare documents for formal publication
  • Archiving: Create archival copies of document collections
  • Analysis: Export for text analysis or data mining
  • Sharing: Distribute documents to colleagues or other researchers
  • Integration: Connect document data with other systems
  • Backup: Preserve document content and structure
  • Print Production: Prepare documents for physical printing
  • Teaching: Create teaching materials from document collections
  • Transformation: Convert documents to new formats for different uses

Advanced Export Strategies

  • Selective Content: Export only the document components you need
  • Staged Exports: Export different components in separate operations
  • Format Combinations: Use multiple formats for different aspects of your data
  • Template-Based: Apply custom templates to standardize output appearance
  • Multi-Format: Generate multiple formats in a single export operation
  • Incremental: Export only documents that have changed since last export

Best Practices

  • Define clear criteria for document selection before starting export
  • Choose the format that best matches your intended use
  • Include sufficient metadata to provide context for documents
  • Test your export configuration with a small document set first
  • Document your export parameters for reproducibility
  • Consider file size limitations for large document collections
  • Verify the integrity and completeness of exported documents
  • Include documentation explaining document structure and content
