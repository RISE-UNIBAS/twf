Import Dictionaries

This page allows you to import dictionary data from external files into your project. Dictionary imports help you quickly populate your dictionaries with pre-existing data, saving time and ensuring consistency.

Import Sources

You can import dictionary data from:

  • JSON files exported from another Transkribus Workflow project
  • CSV files with properly formatted dictionary entries
  • Excel files with structured dictionary data
  • Standard dictionary exchange formats

Import Process

The dictionary import process involves several steps:

  1. Select the file format and source file
  2. Configure how to map fields from the source to your dictionary structure
  3. Specify handling for duplicates and conflicts
  4. Preview the import results
  5. Confirm and process the import
  6. Review the imported entries

File Format Requirements

Depending on the file format you're importing, there are specific requirements:

JSON Format

For JSON imports, the file should contain a structured array of dictionary entries, with each entry including:

  • Required fields for your dictionary type (label, type, etc.)
  • Optional metadata and additional properties
  • Properly formatted nested structures if applicable

JSON exported from another Transkribus Workflow project will already have the correct structure.

CSV/Excel Format

For tabular formats, your file should:

  • Include a header row with column names
  • Contain the required fields for dictionary entries
  • Use consistent formatting for all entries
  • Use a standard delimiter (comma or tab)

Handling Duplicates

When importing, you can choose how to handle duplicate entries:

  • Skip: Keep the existing entry and ignore the imported one
  • Replace: Overwrite the existing entry with the imported one
  • Merge: Combine information from both entries
  • Create New: Add the imported entry as a new entry (possibly with a modified label)

Mapping Fields

The import tool allows you to map fields from your source file to the appropriate dictionary fields:

  • Match source columns/fields to dictionary fields
  • Configure transformations for field values if needed
  • Set default values for missing fields
  • Handle special fields like normalized forms or external IDs

Best Practices

  • Prepare your import file carefully, ensuring consistency and completeness
  • Start with a small test import to verify your configuration
  • Back up your project before large imports
  • Document your import process and sources for future reference
  • Review imported entries to ensure quality and consistency
  • Use consistent naming conventions across imported dictionaries
