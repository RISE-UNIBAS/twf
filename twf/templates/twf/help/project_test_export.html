Test Export

The Test Export feature allows you to request and process a single page from your Transkribus collection to verify that the export and parsing functions work correctly before committing to a full collection export.

Feature Status

Note: This feature is currently marked as "Not Implemented Yet" (NIY). While the interface elements are visible, the functionality is still under development and may not work as expected.

Purpose

Test Export is designed to help you:

  • Verify your Transkribus connection and authentication credentials
  • Check that the system can properly retrieve data from your collection
  • Preview how your Transkribus data will be processed and displayed
  • Identify potential issues with your collection structure before processing the entire dataset
  • Test export settings and configurations on a small sample

How to Use (When Implemented)

  1. Prerequisites:
    • Ensure your Transkribus credentials are configured in the {% help_url 'project_settings_credentials' 'Credential Settings' %}
    • Verify your project is associated with a valid Transkribus collection ID
  2. Request Test Page:
    • Click the "Request Test Page" button
    • The system will connect to Transkribus and request a single page from your collection
    • The status of your request will be displayed in the "Current status" section
  3. Review Results:
    • When implemented, this page will show parsing results and metadata extraction from the test page
    • You'll be able to review text recognition, layout analysis, and metadata
    • This preview will help you understand how your data will be processed in a full export

Benefits of Test Export

  • Time-Saving: Identify issues before processing large collections
  • Configuration Testing: Verify that your export settings produce the expected results
  • Troubleshooting: Easier to diagnose problems with a single page than with a full collection
  • Quality Control: Ensure data quality and integrity before full processing

What to Look For (When Implemented)

When reviewing a test export, pay attention to:

  • Text recognition accuracy
  • Layout recognition and region identification
  • Metadata extraction and formatting
  • Character encoding and special character handling
  • Image quality and resolution
  • Tag and annotation extraction

Alternative Approaches

Until this feature is fully implemented, you can:

  • Use the full {% help_url 'project_tk_export' 'Request Transkribus Export' %} feature and examine the results of a small collection
  • Create a test collection in Transkribus with just a few pages for export testing
  • After exporting, check individual documents in the Document view to verify proper data extraction
