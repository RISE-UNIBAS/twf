Import/Export Overview

The Import/Export section provides comprehensive tools for importing external data into your project and exporting your project data in various formats. These features enable data exchange, publication, preservation, and integration with other systems and workflows.

Export Capabilities

The system offers several export options to meet different needs:

  • Export Documents:
    • Export selected documents in various formats
    • Configure document content, metadata, and structure options
    • Apply filters to select documents based on specific criteria
    • Generate human-readable or machine-processable outputs
    • Include or exclude related content (tags, annotations, etc.)
  • Export Collections:
    • Export curated collections while preserving their organization
    • Include collection structure, item relationships, and metadata
    • Export collection content along with collection information
    • Generate collection-specific outputs for sharing or publication
    • Create portable representations of your organized research materials
  • Export Dictionaries:
    • Export terminology dictionaries with normalized forms
    • Include dictionary structure, entries, and relationships
    • Preserve connections to external authority records
    • Generate specialized vocabulary resources from your dictionary data
    • Create standard-compliant terminology exports
  • Export Tags:
    • Export tags and annotations from across your project
    • Include tag types, contexts, and relationships
    • Generate structured representations of entity mentions
    • Create datasets for entity-focused analysis
    • Preserve connections between tags and dictionary entries
  • Export Project:
    • Export your entire project as a comprehensive package
    • Include all documents, collections, dictionaries, and settings
    • Create complete backups or archives of your research data
    • Generate migration-ready packages for system transfers
    • Preserve the full context and structure of your project
  • Export to Zenodo:
    • Publish your research data to the Zenodo open science repository
    • Obtain DOIs (Digital Object Identifiers) for citation and reference
    • Meet open data requirements for funding agencies
    • Create public or controlled access to your research materials
    • Ensure long-term preservation of your research outputs

Import Capabilities

The system also provides import functionality:

  • Import Dictionaries:
    • Import terminology dictionaries from external sources
    • Integrate vocabulary resources into your project
    • Map imported dictionary structures to your project's needs
    • Incorporate external terminology standards
    • Merge dictionaries from multiple sources
  • Import Metadata:
    • Import document and page metadata from structured sources
    • Map external metadata schemas to your project structure
    • Enhance your documents with external descriptive information
    • Integrate catalog or archival metadata with your collection

Export Configuration

The export system can be customized through:

  • Export Configuration:
    • Configure global export settings for your project
    • Define default export formats and parameters
    • Create custom export templates for different purposes
    • Set up metadata field mappings and transformations
    • Configure file structure and naming conventions
    • Define validation rules and quality controls

Export Management

The Export Overview page provides several management features:

  • Recent Exports:
    • View a list of your recent export operations
    • See export status, completion time, and size
    • Download completed exports directly from the overview
    • Track export history across your project
  • Export Management:
    • Delete unnecessary or outdated exports
    • Rerun previous exports with the same parameters
    • View detailed information about each export
    • Manage export storage and organization

Export Formats

The system supports various export formats to meet different needs:

  • Data Formats:
    • JSON: Structured data with complete relation preservation
    • XML: Structured markup with schema validation
    • CSV/Excel: Tabular formats for spreadsheet analysis
    • RDF: Linked data format for semantic web applications
    • TEI: Text Encoding Initiative format for humanities scholarship
  • Publication Formats:
    • PDF: Portable document format for human reading
    • HTML: Web-ready format for online presentation
    • DOCX/ODT: Word processor formats for editing
    • EPUB: Electronic publication format for e-readers
  • Archival Formats:
    • ZIP/TAR: Compressed archives for complete projects
    • METS/MODS: Metadata encoding and transmission standards
    • BagIt: Hierarchical packaging format for digital preservation

Common Export Workflows

Typical export workflows include:

  • Research Publication: Exporting content for academic publication
  • Data Preservation: Creating archival packages for long-term storage
  • Collaboration: Sharing project data with research partners
  • Analysis: Exporting structured data for computational analysis
  • Presentation: Creating publication-ready documents for sharing
  • Integration: Exporting data for use in other systems
  • Migration: Moving projects between different environments
  • Backup: Creating comprehensive project backups

Best Practices

  • Configure export templates for your common export scenarios
  • Document your export parameters for reproducibility
  • Test exports with small datasets before exporting large collections
  • Consider your audience's needs when selecting export formats
  • Include sufficient documentation with your exports
  • Follow relevant standards for your field when configuring exports
  • Periodically review and clean up your export history
  • Verify export integrity after completion
