Context Help: Export Tags

The Export Tags page allows you to export tag data from your project in various formats. Tags represent annotations and markup present in your documents, often linked to dictionary entries for standardization.

Export Process

  1. Select Tags: Choose which tags to include in your export:
    • Filter by tag type (person, place, date, etc.)
    • Filter by status (resolved, open, parked, ignored)
    • Filter by dictionary association
  2. Choose Format: Select the format for your export (JSON, CSV, XML, etc.).
  3. Configure Options: Set specific export settings:
    • Include tag context (surrounding text)
    • Include document/page references
    • Include dictionary information
    • Include normalized forms
  4. Start Export: Click the export button to begin the process.

Tag Data Options

When exporting tags, you can include:

  • Tag Text: The actual text content of each tag.
  • Tag Type: The category or type classification of each tag.
  • Context: Text surrounding the tag for context.
  • Location: Document and page where each tag appears.
  • Position: Precise coordinates or positions within documents.
  • Normalization: Standardized forms linked through dictionaries.
  • Metadata: Additional information associated with tags.

Export Use Cases

  • Data Analysis: Export tags for statistical analysis in external tools.
  • Visualization: Create visual representations of tag distributions.
  • Entity Networks: Build relationship networks between tagged entities.
  • Quality Control: Review tag consistency and accuracy.
  • Publication: Include structured tag data in research publications.
