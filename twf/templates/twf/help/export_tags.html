Export Tags

The Export Tags page allows you to export tag data from your project in various formats. Tags represent annotations and markup present in your documents, including entities such as people, places, dates, events, and other elements that have been identified and potentially linked to dictionary entries for standardization.

Export Process

  1. Select Tags: Choose which tags to include in your export:
    • Filter by tag type (person, place, date, organization, event, etc.)
    • Filter by status (resolved, open, parked, ignored)
    • Filter by dictionary association (linked, unlinked, specific dictionaries)
    • Filter by document or collection
    • Filter by date range of creation or modification
    • Filter by text content or pattern matching
    • Filter by normalization status
  2. Choose Format: Select the format for your export:
    • JSON (comprehensive, preserves relationships)
    • CSV (tabular format for spreadsheet analysis)
    • XML (structured format with schema options)
    • TEI (Text Encoding Initiative format for scholarly encoding)
    • RDF (Resource Description Framework for linked data)
    • Custom formats configured in project settings
  3. Configure Options: Set specific export settings:
    • Include tag context (surrounding text with configurable window size)
    • Include document/page references with detail level options
    • Include dictionary information and normalization data
    • Include relationship data between tags
    • Configure character encoding and special character handling
    • Set hierarchical structure for nested tags
    • Configure metadata fields to include
  4. Preview: Generate a sample of the export for verification
  5. Start Export: Click the export button to begin the process
  6. Monitor Progress: For large tag exports, track the progress
  7. Download: Save the completed export to your local system or server location

Tag Data Options

When exporting tags, you can include various data components:

  • Tag Text: The actual text content of each tag in the document
  • Tag Type: The category or type classification of each tag
  • Context: Text surrounding the tag for contextual understanding
  • Location: Document, page, and position where each tag appears
  • Coordinates: Precise coordinates or positions within documents
  • Normalization: Standardized forms linked through dictionaries
  • Dictionary Links: Connections to dictionary entries
  • External Identifiers: Links to external authority records (GND, Wikidata, etc.)
  • Metadata: Additional information associated with tags
  • Creation Info: When and by whom the tag was created
  • Modification History: Changes made to tags over time
  • Relationships: Connections between tags (co-occurrence, hierarchy, etc.)
  • Status Information: Current workflow status and processing state

Export Format Details

Different export formats offer various capabilities:

  • JSON:
    • Preserves complex relationships and hierarchies
    • Good for programmatic processing
    • Maintains all metadata and structure
    • Easily parsed by most programming languages
  • CSV:
    • Ideal for tabular data analysis
    • Compatible with spreadsheet applications
    • Good for statistical processing
    • Limitations with complex hierarchical data
  • XML/TEI:
    • Structured representation with schema validation
    • Good for scholarly interchange
    • Supports complex tag hierarchies
    • Compatible with digital humanities tools
  • RDF:
    • Supports semantic web applications
    • Enables sophisticated linked data representations
    • Good for integration with knowledge graphs
    • Supports standardized ontologies and vocabularies

Export Use Cases

  • Data Analysis: Export tags for statistical analysis in external tools
  • Visualization: Create visual representations of tag distributions and networks
  • Entity Networks: Build relationship networks between tagged entities
  • Quality Control: Review tag consistency and accuracy across documents
  • Publication: Include structured tag data in research publications
  • Machine Learning: Create training data for NLP and other ML applications
  • Integration: Connect tag data with external databases and systems
  • Corpus Building: Contribute to larger annotated text corpora
  • Collaborative Research: Share standardized entity data with colleagues
  • Knowledge Base Creation: Build specialized domain knowledge bases

Best Practices

  • Define clear selection criteria before exporting to ensure data relevance
  • Choose export formats based on your intended downstream use
  • Include sufficient contextual information for tags to be meaningful
  • Document your export parameters for reproducibility
  • Consider file size limitations when exporting large tag collections
  • Verify a sample of exported data to ensure quality and completeness
  • Include relevant metadata to maintain the provenance of your tag data
